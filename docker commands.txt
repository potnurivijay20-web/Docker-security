## 🔹 Installation (Linux)
```bash
# Official convenience script (Ubuntu/Debian/RHEL/Fedora)
curl -fsSL https://get.docker.com | sudo sh

# Start and enable Docker
sudo systemctl start docker && sudo systemctl enable docker

# Add user to docker group (avoid sudo)
sudo usermod -aG docker $USER && newgrp docker
```

---

## 🔹 Basic Commands – Images & Containers
```bash
# Pull an image from a registry

docker pull ubuntu:22.04

# List local images
docker images

# Remove an image
docker rmi ubuntu:22.04

# Run a container (interactive, remove when stopped)
docker run -it --rm ubuntu:22.04 bash

# Run in detached mode, give it a name

docker run -d --name my_container nginx:alpine

# List running containers
docker ps

# List all containers (including stopped)
docker ps -a

# Stop, start, restart a container
docker stop my_container
docker start my_container
docker restart my_container

# Remove a stopped container
docker rm my_container

# Force remove a running container
docker rm -f my_container
```

---

## 🔹 Interacting with Containers
```bash
# Execute a command inside a running container
docker exec -it my_container bash

# Run a new process in a container
docker exec my_container cat /etc/os-release

# View logs (follow)
docker logs -f my_container

# Copy file from host to container
docker cp ./file.txt my_container:/tmp/

# Copy file from container to host
docker cp my_container:/tmp/file.txt ./
```

## 🔹 Building Images – Dockerfile
```bash
# Build an image from a Dockerfile in the current directory
docker build -t myapp:1.0 .

# Build with a specific Dockerfile
docker build -f Dockerfile.prod -t myapp:prod .

# Build without using cache
docker build --no-cache -t myapp:latest .
```

---

## 🔹 Tagging & Registries
```bash
# Tag an image for a repository
docker tag myapp:1.0 myusername/myapp:1.0

# Push to Docker Hub (login first)
docker login
docker push myusername/myapp:1.0

# Pull from a private registry
docker login myregistry.com
docker pull myregistry.com/myapp:1.0
```

---

## 🔹 Volumes & Data Persistence
```bash
# Bind mount (host directory to container)
docker run -v /host/data:/container/data nginx

# Named volume (Docker-managed)
docker volume create my_data
docker run -v my_data:/container/data nginx

# List volumes
docker volume ls

# Remove unused volumes
docker volume prune
```

---

## 🔹 Networking
```bash
# List networks
docker network ls

# Create a user-defined bridge network
docker network create my_net

# Run container on a specific network
docker run --net my_net --name web nginx

# Connect a running container to a network
docker network connect my_net my_container

# Inspect a network (see connected containers)
docker network inspect my_net
```

## 🔹 Environment & Configuration
```bash
# Set environment variable
docker run -e MY_VAR=hello nginx

# Use an env-file
docker run --env-file ./prod.env nginx

# Set restart policy
docker run --restart unless-stopped nginx

# Limit resources (CPU/memory)
docker run --memory 256m --cpus 1 nginx
```

## 🔹 Inspecting & Debugging
```bash
# Full low-level info about container/image
docker inspect my_container

# View container’s logs
docker logs my_container

# See running processes inside a container
docker top my_container

# Live resource usage stats
docker stats

# View events stream
docker events

# Check the history of an image
docker history myapp:1.0
```

---

## 🔹 System Cleanup & Maintenance
```bash
# Remove all stopped containers
docker container prune

# Remove all unused images
docker image prune -a

# Remove everything not in use (containers, images, volumes, networks)
docker system prune -a --volumes

# Show disk usage by Docker objects
docker system df
```

## 🔹 Multi‑Container Apps – Docker Compose
```bash
# Start services (detached)
docker-compose up -d

# Build and start
docker-compose up --build -d

# Stop services
docker-compose down

# Remove volumes as well
docker-compose down -v

# View running services
docker-compose ps

# View logs of all services
docker-compose logs -f

# Execute a command in a service container
docker-compose exec web bash
```

## 🔹 Advanced – Swarm Mode (Orchestration)
```bash
# Initialize a swarm (manager node)
docker swarm init

# Join a worker node (token from manager)
docker swarm join --token <token> <manager-ip>:2377

# Deploy a stack from a compose file
docker stack deploy -c docker-compose.yml my_stack

# List services in the stack
docker stack services my_stack

# List nodes
docker node ls

# Scale a service
docker service scale my_stack_web=5

# Leave swarm
docker swarm leave --force
```

## 🔹 Advanced – Image Management & Distribution
```bash
# Save an image to a tar archive
docker save -o myimage.tar myapp:1.0

# Load an image from a tar archive
docker load -i myimage.tar

# Export a container’s filesystem as tar
docker export my_container > container.tar

# Import a tar to create an image
docker import container.tar my-new-image:1.0

# Create image from container changes (commit)
docker commit my_container myapp:manual

# Push to a registry with a specific digest
docker push myregistry.com/myapp@sha256:abc123...
```

```bash
# Scan an image for vulnerabilities (requires Docker Scout)
docker scout quickview myapp:1.0

# Detailed CVE report
docker scout cves myapp:1.0
```

## 🔹 One‑Liners & Useful Snippets
```bash
# Stop all running containers
docker stop $(docker ps -q)

# Remove all containers
docker rm -f $(docker ps -aq)

# Delete all images
docker rmi -f $(docker images -q)

# Enter the last created container’s shell
docker exec -it $(docker ps -lq) bash

# See IP address of a container
docker inspect -f '{{range.NetworkSettings.Networks}}{{.IPAddress}}{{end}}' my_container
```
